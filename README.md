# Minecraft Server Dashboard API

Minecraft サーバーのライフサイクルを管理する REST API です。
複数の Organization が同一インスタンス上でサーバーを独立して運用できます。

## 主な機能

- **サーバー管理** — 作成・起動・停止・再起動・設定編集。vanilla / paper / forge / fabric など主要サーバータイプに対応
- **Organization / アクセス制御** — Organization 単位でリソースを分離。メンバーごとに機能単位のパーミッションを付与できる
- **バックアップ / 復元** — 手動・スケジュール実行。ローカルまたは S3 互換ストレージへの保存に対応
- **プレイヤーグループ** — OP リストとホワイトリストを Organization 内で共有し、複数サーバーに一括適用
- **ファイル管理** — API 経由でサーバーディレクトリのファイルを読み書き。編集履歴とロールバックに対応
- **リアルタイム配信** — WebSocket でコンソールログとサーバーステータスをストリーミング
- **監査ログ** — すべての変更系操作を記録。Organization の管理者が閲覧・フィルタできる

## アーキテクチャの特徴

サーバープロセスの実行は **Runner** と呼ぶ抽象化レイヤーに委譲します。
API Core はファイルシステムやコンテナランタイムに直接依存せず、Runner インターフェース経由でのみ操作します。
これにより Docker / Podman / host プロセスといった実行基盤を差し替えられます。

起動・停止・バックアップなどの重い処理はすべて非同期ジョブとして実行され、API は即座に `job_id` を返します。
